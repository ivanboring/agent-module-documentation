# computed_field — agent start

Adds **read-only fields whose value is generated by code**, not stored. Each field's value
comes from a `computed_field` plugin that declares a **core field type** (`string`,
`entity_reference`, …) and implements `computeValue()`; output reuses core field formatters.
Depends only on core `field`. Defines the `computed_field` plugin type and a `computed_field`
config entity. Admin UI lives in the **computed_field_ui** submodule (documented separately).
Ships one usable plugin: `reverse_entity_reference`. Permission: `administer computed_field
entities`. No Drush commands.

- Settings key (field name prefix) + the `computed_field` config entity that represents a configured field → [configure/settings.md](configure/settings.md)
- Write a computed field plugin (attribute, field type, `computeValue()`, attach modes, cacheability, configurable plugins) → [plugins/computed-field-plugins.md](plugins/computed-field-plugins.md)
- Create/inspect computed fields in code (manager service, config entity, `reverse_entity_reference`) + the info-alter hook → [api/programmatic.md](api/programmatic.md)
