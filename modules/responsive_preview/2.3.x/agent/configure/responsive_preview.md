# Configure — responsive_preview

Devices are `responsive_preview_device` **config entities** (`ConfigEntityType`, config prefix
`device`, so config files are named `responsive_preview.device.{id}.yml`). Manage them in the UI at:

- Collection / config route (`configure`): `entity.responsive_preview_device.collection`
  → `/admin/config/user-interface/responsive-preview`
- Add: `entity.responsive_preview_device.add_form` → `/admin/config/user-interface/responsive-preview/add`
- Edit: `entity.responsive_preview_device.edit_form` → `.../{responsive_preview_device}/edit`
- Delete: `entity.responsive_preview_device.delete_form` → `.../{responsive_preview_device}/delete`

Routes are generated by the entity's `route_provider` (`DeviceHtmlRouteProvider`); there is **no
`responsive_preview.routing.yml`**. Admin permission for all of them is `administer responsive preview`.
The menu link lives under `system.admin_config_ui`; an "Add Device" action link appears on the collection.

## Device entity fields (config_export / schema)

| Key | Type | Notes |
|---|---|---|
| `id` | string | machine name |
| `label` | label | human name shown in the toolbar list |
| `weight` | integer | sort order in the list (lower = first) |
| `status` | integer (0/1) | 1 = "Show device in list" (only `status=1` devices render) |
| `orientation` | string | `portrait` or `landscape` |
| `dimensions.width` | integer | px |
| `dimensions.height` | integer | px |
| `dimensions.dppx` | float | dots per pixel (1 = classic, 2–3 = HiDPI) |
| `langcode` | string | default language |

## Default devices (config/install)

| id | label | width×height | dppx | orientation | weight |
|---|---|---|---|---|---|
| `phone` | Phone | 1170×2532 | 3 | portrait | 0 |
| `tablet_portrait` | Tablet Portrait | 1640×2360 | 2 | portrait | 1 |
| `tablet_landscape` | Tablet Landscape | 2360×1640 | 2 | landscape | 2 |
| `desktop` | Desktop | 1920×1080 | 1 | landscape | 3 |

The edit form (`DeviceForm`) exposes Label, machine id, "Show device in list" (status), Width,
Height, Dots per pixel, and Default orientation. Width/height/dppx are required, min 1.

## Drush / config

Export/inspect a device with core config commands (no module-specific drush):

```bash
drush config:get responsive_preview.device.phone
drush config:set responsive_preview.device.phone status 0 -y   # hide Phone from the list
```

Add a custom device by importing a `responsive_preview.device.{id}.yml` file matching the schema
above, or via the Add form. Which devices appear in the toolbar is purely `status: 1`.
