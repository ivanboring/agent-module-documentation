# block_class — agent start

Adds custom CSS classes, HTML attributes, and a custom ID to any block via the block
config form. Depends on core `block`. Values applied through `hook_preprocess_block()`.
Config UI: **Admin → Config → Content → Block Class** (`block_class.settings`).

- Global settings (field type, limits, autocomplete, attributes, ID replacement) → [configure/settings.md](configure/settings.md)
- Admin list pages + bulk add/rename/delete of classes & attributes → [configure/bulk-operations.md](configure/bulk-operations.md)
- Helper service & how classes/attributes reach markup → [api/service.md](api/service.md)
- Permission → [permissions/permissions.md](permissions/permissions.md)
