Adds meta tags to Views pages/displays, so listing, search, and archive pages built with Views get proper SEO tags.

---

Views-driven pages (listings, archives, search results) are often key landing pages but aren't entities, so they don't inherit entity meta tags. This submodule integrates Metatag with Views — typically as a display extender — letting you configure title, description, and social tags per Views display, with token support (including View argument tokens). Depends on the main Metatag module and core Views. Useful for giving category, tag, and search pages accurate, indexable metadata.

---

- Add a title/description to a Views listing page.
- Set Open Graph tags on a Views archive page.
- Give category/term listing pages proper SEO metadata.
- Provide meta tags per Views display (page, feed, etc.).
- Use View argument tokens in meta tags.
- Control robots indexing on a listing page.
- Set canonical URLs on Views pages.
- Provide social-share previews for search-results pages.
- Override global defaults for a specific Views display.
- Keep Views metadata in exportable config.
- Standardize SEO across all listing pages.
- Differentiate metadata between multiple displays of one View.
- Add descriptions to blog/news index pages.
- Improve indexing of faceted/filtered listing pages.
- Provide meta tags for a Views-based front page.
- Combine with Open Graph/Twitter submodules on listings.
