CSV Importer bulk-creates or updates any Drupal content entity (nodes, users, taxonomy terms, media, etc.) from an uploaded CSV file, mapping CSV columns to entity fields with no config required.

---

CSV Importer adds an admin form at `/admin/content/csv-importer` (Content > Import CSV) where you pick a content entity type, its bundle (when it has one), a delimiter, and upload a UTF-8 CSV. The first CSV row holds the target field machine names; each following row becomes one entity. It uses a single derived "Importer" plugin (`importer:<entity_type>`) generated by a deriver for every content entity type, runs the work through the Batch API, and requires the CSV to contain every field the bundle marks required (missing ones are reported and the import aborts). A header cell may target multiple field properties by piping names (`body|value|format`), and a cell value wrapped as `values(a+b+c)` or `multiple(a+b+c)` becomes a multi-value field. Cell values that are stream-wrapper URIs (`public://`, `private://`) or `http(s)` URLs are downloaded and saved as managed files, replaced by their file IDs; bare local paths are rejected for security. If a row supplies the entity id key the matching entity is loaded and updated (and translated when a `langcode` column is present) instead of created. Every import is logged to a `csv_importer_history` database table exposed at `.../csv-importer/history`, from which an import can be reverted (deleting the entities it created). Extension points are the `importer` plugin type, the `csv_importer.parser` service, the `hook_csv_importer_pre_import()` alter hook, and the `importer_info` plugin-info alter hook. The module has no configuration UI, no config entities, and no Drush commands.

---

- Bulk-create nodes of a given content type from a spreadsheet exported to CSV.
- Bulk-create taxonomy terms in a vocabulary from a CSV list.
- Bulk-create users from a CSV of names/emails.
- Import media entities or any custom content entity type from CSV.
- Update existing entities in bulk by including their id column (e.g. `nid`, `uid`, `tid`) in the CSV.
- Map a CSV column to a specific field property using the `field|property` pipe header (e.g. `body|value`, `body|format`).
- Populate multi-value fields from one cell using `values(a+b+c)` or `multiple(a+b+c)` syntax.
- Attach images/files to entities by putting `public://path/file.jpg` or an `https://…` URL in the CSV cell.
- Attach several files to one multi-value file/image field via the `values(...)` / `multiple(...)` syntax with URIs.
- Import translated content by adding a `langcode` column so rows update or add translations of existing entities.
- Choose a non-comma delimiter (`,`, `~`, `;`, `:`) when your data contains commas.
- Enforce that required fields are present before any entity is written (validation lists missing fields).
- Migrate content between Drupal sites by exporting to CSV and re-importing.
- Seed a development or staging site with realistic bulk content quickly.
- Load reference/lookup data (e.g. product catalogs, glossaries) into content entities.
- Run large imports without timeouts thanks to Batch API processing.
- Review a history log of every import (file, entity type, count, date, status) at the history page.
- Undo a mistaken import by reverting it, which deletes the entities that import created.
- Preprocess/normalize incoming CSV data programmatically via `hook_csv_importer_pre_import()`.
- Parse a CSV file in custom code with the `csv_importer.parser` service (`getCsvById`, `getCsvFieldsById`).
- Define a custom importer behavior by extending `ImporterBase` for a specific entity type plugin.
- Gate CSV import access with the dedicated `access csv importer` permission (restricted).
- Programmatically kick off an import by creating an `importer:<entity_type>` plugin instance and calling `process()`.
- Alter the discovered importer plugin definitions via the `importer_info` alter hook.
- Import content for multiple fields sharing one source value using multi-target pipe headers.
