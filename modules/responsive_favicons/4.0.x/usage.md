Responsive Favicons injects the full set of device favicon tags (iPhone, Android, iPad, tablets, desktop) generated by realfavicongenerator.net into your site's HTML head, and serves the icon files from Drupal so they work in multisite setups without polluting the docroot.

---

You generate a favicon package and its HTML `<link>`/`<meta>` snippet at realfavicongenerator.net, then paste the HTML and either upload the `.zip` package or point to an in-repo path on the module's settings form at `/admin/config/user-interface/responsive_favicons`. On every page request the module's `responsive_favicons.manager` service loads the configured tags, normalises their paths, and attaches them to the head via `hook_page_attachments()` (plus a preprocess hook so the maintenance page is covered too). It registers dynamic routes for the well-known icon paths (`/apple-touch-icon.png`, `/site.webmanifest`, `/favicon.svg`, `/browserconfig.xml`, and `/favicon.ico`) so requests to those URLs are delivered from the configured favicon directory by its `GetFile` controller. Icons can be stored per-site in each site's public files directory (upload mode) or referenced from a theme/module path (internal path mode). Options let you add a cache-refresh query suffix, still emit tags when files are missing, and strip Drupal's default core/theme `favicon.ico`. A `hook_requirements()` check reports how many favicon tags were found, flags any missing referenced files, and warns about conflicts with the `favicon` and `pwa` modules. Two alter hooks let other modules modify the loaded tags or the normalised icon path.

---

- Add a complete cross-device favicon set (Apple touch icons, Android/Chrome icons, Windows tiles, SVG icon) to a Drupal site from one realfavicongenerator.net package.
- Paste the realfavicongenerator.net HTML snippet and have the module rewrite relative icon paths to correct, working URLs automatically.
- Upload a favicons `.zip` once through the admin form and let the module unpack and store the icons for you.
- Store per-site favicon variations in each site's public files directory in a Drupal multisite, avoiding icons cluttering the shared docroot.
- Reference favicon files that already live in a theme or module (e.g. `/themes/custom/mytheme/favicons`) via the "Use internal path" mode instead of uploading.
- Serve `/favicon.ico` through Drupal so a multisite can return a different icon per site.
- Serve the well-known icon paths (`/apple-touch-icon.png`, `/apple-touch-icon-precomposed.png`, `/site.webmanifest`, `/browserconfig.xml`, `/favicon.svg`) from the configured directory.
- Remove Drupal's default core/theme `favicon.ico` tag so it does not conflict with your custom icons.
- Add a cache-refresh suffix to icon URLs so browsers pick up updated icons without a manual cache clear.
- Emit favicon tags even when some referenced files are missing, for staged rollouts.
- Get a status-report warning listing favicon tags whose files cannot be found (or carry stray cache-busting parameters), so misconfiguration is visible.
- See a status-report count confirming how many favicon tags the module is emitting.
- Get warned on the status report when the redundant `favicon` module is also enabled.
- Get warned when the `pwa` module is active and a conflicting web manifest has been declared.
- Ensure favicons also appear on the site maintenance/offline page, not just normal pages.
- Let another module programmatically alter the loaded favicon tags via `hook_responsive_favicons_tags_alter()`.
- Let another module rewrite the normalised icon path via `hook_responsive_favicons_icon_path_alter()`.
- Gate favicon administration behind the `administer responsive favicons` permission for a restricted editor role.
- Replace an inferior single low-res `favicon.ico` with high-quality, size-specific icons for retina and mobile home screens.
- Clean up favicon files automatically on uninstall (the configured upload directory is deleted).
- Recursively remove previously uploaded favicon files when uploading a fresh package via the "Remove previously uploaded files" option.
- Avoid route normalisation clashes with the `redirect` module (favicon routes opt out of the normalizer automatically).
- Export the module's settings (`responsive_favicons.settings`) as configuration to deploy identical favicon setup across environments.
