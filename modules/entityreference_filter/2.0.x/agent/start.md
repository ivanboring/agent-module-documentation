# Views Reference Filter (entityreference_filter)

Adds a Views filter whose exposed dropdown options are generated by running a *separate*
**Entity Reference** view display — so you curate which entities appear, and can build
dependent (cascading, AJAX) exposed filters. Registers one plugin,
`entityreference_filter_view_result` (extends core `ManyToOne`), attached by
`hook_views_data_alter()` to entity-id columns (nid/uid/tid) and entity-reference
(`*_target_id`) fields as extra filters labelled "{Field} (entityreference filter)". No
admin UI / `configure` route — everything is configured on the filter handler inside the
Views UI. Requires core `views` + `field`. Release 2.0.0-beta7 (**beta**).
