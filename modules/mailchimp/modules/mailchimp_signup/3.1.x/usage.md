Mailchimp Signup provides configurable newsletter signup forms — exposed as blocks and/or standalone pages — that let anonymous and authenticated visitors subscribe to one or more Mailchimp audiences.

---

This submodule adds a `mailchimp_signup` config entity representing a reusable signup form, managed at `/admin/config/services/mailchimp/signup`. Each signup form chooses one or more target audiences, the merge fields to collect, its display mode (a block, a dedicated page, or both), and messaging. Block forms are exposed through a block plugin (`MailchimpSignupSubscribeBlock`) you can place in any region; page forms get their own route generated by `MailchimpSignupRoutes`. Submitting the form subscribes the visitor to the selected audience(s), honoring double opt-in and the confirmation message configured on the base module. Access to signup pages is gated by the `access mailchimp signup pages` permission, while form management requires `administer mailchimp signup entities`. It is the quickest way to add "join our mailing list" functionality to a site without custom form code.

---

- Add a newsletter signup block to the sidebar or footer.
- Create a dedicated "Subscribe" landing page.
- Let visitors subscribe to multiple audiences from one form.
- Collect merge fields (name, interests) during signup.
- Expose the same signup as both a block and a page.
- Place different signup blocks on different sections of the site.
- Require double opt-in so subscribers confirm via email.
- Show a custom confirmation message after signup.
- Restrict who can view standalone signup pages via permission.
- Let anonymous visitors join a mailing list without an account.
- Offer interest-group choices on the signup form.
- Build several distinct signup forms for different campaigns.
- Deploy signup form configuration between environments as config.
- Style signup forms with theme templates provided by the module.
- Add a subscribe call-to-action to any landing page via block placement.
- Route page signups through their own generated URL.
- Manage all signup forms from a single admin listing.
- Reuse one configured signup form across multiple pages.
