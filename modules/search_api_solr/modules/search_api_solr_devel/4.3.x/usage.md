A development aid that logs and exposes the raw Solr requests and responses behind Search API Solr, making it easy to debug indexing and query behavior.

---

Search API Solr Devel integrates with the Devel module to surface what actually happens between Drupal and Apache Solr. It logs the Solarium requests and responses (via a logging layer in `src/Logging/`) and adds a Devel tab/controller so you can inspect the exact queries, parameters, and returned documents for an index or server. This is invaluable when a search returns unexpected results, when relevance tuning isn't behaving, or when you need to confirm which fields and boosts are being sent to Solr. It is a developer-only tool that depends on both `devel` and `search_api_solr`, and should not be enabled on production. There is no configuration UI — enabling the module is enough to start capturing debug output.

---

- Inspect the raw Solr query sent for a search.
- View the raw Solr response and returned documents.
- Debug why a query returns unexpected results.
- Confirm which fields and boosts are sent to Solr.
- Verify filter queries (`fq`) generated by facets.
- Check parameters produced by processors and hooks.
- Log Solarium requests during development.
- Trace indexing requests item-by-item.
- Compare query output before/after an alter hook.
- Validate multilingual field mapping per language.
- Diagnose highlighting/snippet parameters.
- Examine "more like this" query construction.
- Review streaming expression requests.
- Troubleshoot connector/endpoint issues locally.
- Teach new developers how Search API maps to Solr.
- Confirm commit/finalize behavior during indexing.
- Spot malformed queries from custom code.
- Capture Solr responses for bug reports.
- Debug spellcheck/autocomplete queries.
- Verify pagination and sort parameters.
