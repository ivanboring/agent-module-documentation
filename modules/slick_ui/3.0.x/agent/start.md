# slick_ui — agent start

Admin UI submodule for **slick**. No runtime/rendering logic of its own — it only provides the
management forms for the `slick` optionset config entity (defined by the parent module).
Depends on `slick:slick`. All screens require the `administer slick` permission.

- Manage optionsets & the settings page (routes, forms, links) → [configure/optionsets.md](configure/optionsets.md)
- Permission it defines (`administer slick`) → [permissions/permissions.md](permissions/permissions.md)
