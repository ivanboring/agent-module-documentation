# Permissions

Static permissions in `entityqueue.permissions.yml`:

| Permission | Gates |
|---|---|
| `administer entityqueue` | Full admin: create/update/delete all queues and configuration. `restrict access` (trusted). |
| `manipulate entityqueues` | Access the Entityqueues list. |
| `manipulate all entityqueues` | Update the items of any queue. |

Dynamic per-queue permissions are generated by
`Drupal\entityqueue\EntityQueuePermissions::permissions` (declared via `permission_callbacks`).
For each queue `<id>` you get:

| Permission | Gates |
|---|---|
| `update <id> entityqueue` | Manipulate items of that specific queue. |
| `create <id> entityqueue` | (Multiple-subqueue queues) add subqueues. |
| `delete <id> entityqueue` | (Multiple-subqueue queues) delete subqueues. |

This lets you give an editor rights to just one queue (e.g. `update homepage_featured
entityqueue`) without granting site-wide access. Route access uses the `+`-OR form, e.g.
`administer entityqueue+manipulate entityqueues+manipulate all entityqueues`.
