# Configure the embedded "Global: View" field

The module adds no admin pages. All configuration happens inside the Views UI when editing a
view. There is one field plugin (`@ViewsField("view")`, class
`Drupal\views_field_view\Plugin\views\field\View`), surfaced as **Global: View**.

## Setup steps

1. **Build a child view** with one or more **contextual filters (arguments)** — these receive
   the values the parent passes. Give it a display (e.g. a block/default) to embed. No special
   settings are required beyond the arguments; validators / "argument not present" behavior are
   optional.
2. **Edit the parent view** and add the field **Global: View** (Add → Fields → "View").
3. Save the field once; then reopen it — the **View** and **Display** selects populate
   (display selection needs the view chosen and saved first).
4. Place the Global: View field **below** any field you will use as an argument source, because
   tokens only resolve for fields rendered *before* this one. Rearrange fields if needed.
5. Fill in **Contextual filters** (the `arguments` option) with tokens (below), in the order
   the child view expects them.

## Field options (`config/schema/views_field_view.views.schema.yml`)

Stored on the field handler under `views.field.view`:

| Option | Default | Meaning |
|---|---|---|
| `view` | `''` | Machine name of the child view to embed |
| `display` | `'default'` | Display ID of the child view to render |
| `arguments` | `''` | Token/static list mapped to the child's contextual filters |

These export with the parent view's config (`views.view.*`). Standard field options such as
**hide empty** are inherited from `FieldPluginBase`.

## Argument tokens (field-to-argument mapping)

The `arguments` string is split on commas (`,`) **or** forward slashes (`/`); each part is one
argument passed to the child, in order. Token forms (see the field's "Replacement patterns"
list, generated by `getTokenInfo()`):

| Token | Resolves to |
|---|---|
| `{{ raw_fields.FIELD }}` | Raw value of parent field `FIELD` (`getValue()`) |
| `{{ fields.FIELD }}` | Rendered/last-output value of parent field `FIELD` (whitespace stripped) |
| `{{ raw_arguments.ID }}` | Raw input of parent contextual filter `ID` |
| `{{ arguments.ID }}` | Title of parent contextual filter `ID` |
| any non-token text | Passed through as a **static** argument (e.g. `123` or `"my term"`) |

Notes from source:
- Only fields **up to and including** the View field appear as usable field tokens; fields
  after it are unavailable due to render order.
- Prefer `raw_*` tokens: rendered (`fields` / `arguments`) tokens use the field's last output,
  which may be rewritten markup and cause surprises. If a token has no `raw_`/rendered prefix
  matching, `raw_*` is the effective default.
- Make argument-source fields "clean" (no label/formatting); you may mark them
  **Exclude from display** and still use them as tokens.

## Runtime behavior

At render, each parent row loads the child with `Views::getView()`, checks `access()` on the
chosen display, rewrites the child pager ID per row (so parent/child pagers don't collide),
then `preExecute($args)` + `execute()` and `preview()`. Empty results render nothing when
**hide empty** is set. The Views UI shows an *Edit "…"* link to open the child view.

## Recursion guard & the only global setting

A static `$running` map blocks a view from embedding itself: it renders the literal string
**"Recursion, stop!"**. The single config object `views_field_view.settings` has one key:

| Key | Default | Effect |
|---|---|---|
| `evil` | `false` | When `true`, disables the recursion guard (allows recursive embedding) |

No settings form ships; change it via config only, e.g.
`drush cset views_field_view.settings evil true`. Embedding views adds a query per parent row —
the module's own Views data help advises enabling caching.
