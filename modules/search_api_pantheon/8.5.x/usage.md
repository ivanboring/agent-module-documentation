Search API Pantheon wires Search API and Search API Solr to the Apache Solr add-on that Pantheon provisions per environment, so no manual Solr connection details are ever entered. It ships a ready-to-use "Pantheon Search" server plus a "Primary" index and a Drush toolkit for posting schemas, reloading cores, and diagnosing the connection.

The module defines a `pantheon` SolrConnector plugin that extends Search API Solr's `StandardSolrConnector` and force-overrides its connection settings from Pantheon environment variables (`PANTHEON_INDEX_HOST`, `PANTHEON_INDEX_PORT`, `PANTHEON_INDEX_SCHEME`, `PANTHEON_INDEX_PATH`, `PANTHEON_INDEX_CORE`). Because those variables differ per Pantheon environment (Dev, Test, Live, Multidev) and each environment has its own Solr core, the same exported config follows your code across environments with no per-environment editing. Enabling the module installs the `pantheon_search` Search API server (backend `search_api_solr`, connector `pantheon`, Solr version 9, `skip_schema_check: true`) and a `primary` index. Only one Pantheon server per environment is supported — additional Pantheon-connector servers all point at the same core. Schemas are pushed to Solr with `drush search-api-pantheon:postSchema`, which uploads the config-set generated by your installed Search API Solr version (or a custom/jump-start directory you pass), then automatically reloads the core to prevent schema reversion. Diagnostic Drush commands (`diagnose`, `ping`, `select`, `test-index-and-query`, `view-schema`, `force-cleanup`, `reloadSchema`) verify and repair the connection. Solr must be enabled as a Pantheon add-on and `search: version: 9` (or 8) set in `pantheon.yml`. The module only functions on Pantheon's platform where the `PANTHEON_ENVIRONMENT` variable and index variables are present.

---

Search API Pantheon provides Drupal 10/11 integration with the Apache Solr (8 or 9) add-on hosted on Pantheon's platform. It auto-configures the Solr connection per environment by extending Search API Solr, so you index and search without hand-entering any server host, port, or core.

It reads connection details from Pantheon environment variables at runtime and force-overrides the connector configuration, meaning one exported configuration works unchanged across Dev, Test, Live, and Multidev environments — each of which has its own isolated Solr core. On install it creates a "Pantheon Search" Search API server and a "Primary" index; you add fields, post the schema to Solr with a Drush command (which also reloads the core), and index content. Because Search API Solr does the heavy lifting, all normal Search API workflows (Views search pages, facets, autocomplete) apply on top. The bundled Drush commands cover schema posting, core reload, ping/diagnose health checks, ad-hoc Solr queries, an end-to-end index-and-query smoke test, schema file viewing, and a force cleanup of the Solr contents. A `pantheon.yml` `search: version:` entry and the Pantheon Solr add-on are prerequisites; the module is a no-op off Pantheon.

---

- Add hosted Solr full-text search to a Drupal site running on Pantheon without provisioning or maintaining your own Solr server.
- Auto-configure the Solr host/port/scheme/path/core from Pantheon environment variables so no connection details are entered by hand.
- Keep one exported Search API configuration that works unchanged across Dev, Test, Live, and Multidev environments.
- Get an isolated Solr core per Pantheon environment so indexing on Dev never touches Live's index.
- Use the auto-installed "Pantheon Search" server and "Primary" index as a ready-made starting point.
- Create custom Search API indexes that target the "Pantheon Search" server for specific content types or bundles.
- Post/upload the Solr schema config-set to the Pantheon Solr container with `drush search-api-pantheon:postSchema`.
- Push a known-good jump-start config-set from Search API Solr (`solr8`/`solr9`) as a clean schema baseline.
- Upload a fully custom Solr config directory (custom analyzers, tokenizers, field types) by passing a path to `postSchema`.
- Automatically reload the Solr core after a schema update so the schema does not silently revert.
- Manually reload the core with `drush search-api-pantheon:reloadSchema` when a schema push needs re-applying.
- Diagnose a broken Solr connection end-to-end with `drush search-api-pantheon:diagnose` (alias `sapd`).
- Ping the Solr host to confirm reachability with `drush search-api-pantheon:ping` (alias `sapp`).
- Run a smoke test that indexes one item and immediately queries it with `drush search-api-pantheon:test-index-and-query` (alias `sap-tiq`).
- Run an ad-hoc Solr query from the CLI with `drush search-api-pantheon:select <query>` (alias `saps`) for debugging.
- Inspect an uploaded Solr schema file with `drush search-api-pantheon:view-schema <filename>` (alias `sapvs`).
- Wipe all contents from the Solr core with `drush search-api-pantheon:force-cleanup` (alias `sapfc`) to recover from corruption.
- Upgrade from Solr 8 to Solr 9 by switching `pantheon.yml` and reposting the schema.
- Recover from schema incompatibility after upgrading Search API Solr from 4.2.x to 4.3.x by reposting the schema.
- Build search pages, exposed keyword filters, and relevance-sorted results in Views on top of the Pantheon-backed index.
- Layer facets, autocomplete, and spellcheck (via Search API Solr add-ons) on a Pantheon-hosted index, reposting schema after enabling them.
- Automate schema deploys in CI/CD by running `terminus drush <site>.<env> -- search-api-pantheon:postSchema` after each deploy.
- Export index/server config to YAML with `config:export` so search configuration is version-controlled and deployable.
- Uninstall Drupal core's Search module to avoid confusion when standardizing on Search API + Solr.
- Confirm the Pantheon Solr add-on and `search: version` in `pantheon.yml` are correctly wired before going live.
