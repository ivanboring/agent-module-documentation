# Permissions

Defined in `tvi.permissions.yml` (one static permission + dynamic per-vocabulary
permissions generated by `TaxonomyViewsIntegratorPermissions::permissions`).

| Permission | Gates |
|---|---|
| `administer taxonomy views integrator` | The global TVI settings form (`tvi.global_settings`, `/admin/config/user-interface/tvi`) and global overrides. `restrict access: true` — trusted/administrative. |
| `define view for vocabulary {vid}` | Shows the TVI settings fieldset on that **vocabulary's** edit form, letting the user set the vocabulary-wide View override. Generated once per vocabulary. |
| `define view for terms in {vid}` | Shows the TVI settings fieldset on the **term** edit form for terms in that vocabulary, letting the user set per-term View overrides. Generated once per vocabulary. |

The per-vocabulary permissions are dynamic: TVI iterates all vocabularies and emits a
`define view for vocabulary X` and a `define view for terms in X` pair for each. The
term/vocabulary edit forms only render the TVI fieldset when the current user holds the
matching permission (checked in `tvi_form_taxonomy_term_form_alter` /
`tvi_form_taxonomy_vocabulary_form_alter`).

```
drush role:perm:add editor 'define view for terms in tags'
```
