# Permissions

Permissions are **dynamic**, generated by
`SimpleMenuPermissionsPermissions::get()` (registered via `permission_callbacks`
in `simple_menu_permissions.permissions.yml`). They are **plain machine names
that embed the menu id** — there is no config schema and no settings form.

## Global permission

- **`create new menu`** — title "Create new menu". Set as the `_permission`
  requirement on the `entity.menu.add_form` route by the module's
  `RouteSubscriber`, so it gates creating brand-new menus.

## Per-menu permissions

For **every** menu on the site (id `<id>`, e.g. `main`, `footer`, `admin`), six
permissions are generated (title uses the menu's human label):

- **`view <id> menu in menu list`** — "View *<label>* menu in the menu list".
  Controls whether the menu appears in the menus overview
  (`/admin/structure/menu`), filtered by `MenuListBuilder`.
- **`edit <id> menu`** — "Edit *<label>* menu". Grants access to the menu edit
  form (`entity.menu.edit_form`) via `MenuRoutesAccessChecker`.
- **`delete <id> menu`** — "Delete *<label>* menu". Grants access to the menu
  delete form (`entity.menu.delete_form`).
- **`add new links to <id> menu`** — "Add new links to *<label>* menu". Grants
  access to the add-link form (`entity.menu.add_link_form`).
- **`edit links in <id> menu`** — "Edit links in *<label>* menu". Used by the
  `MenuLinkContentAccessControlHandler` for the `update` operation on links in
  that menu, and by the parent-menu selector decorator.
- **`delete links in <id> menu`** — "Delete links in *<label>* menu". Used by
  the `MenuLinkContentAccessControlHandler` for the `delete` operation on links.

Each per-menu permission declares a config dependency on `system.menu.<id>`, so
it is removed automatically if that menu is deleted. A fresh set of six
permissions appears automatically whenever a new menu is created.

## Assigning them

There is **no configuration UI**. Grant the permissions on
**People → Permissions** (`/admin/people/permissions`), or with drush/config:

```php
$role = \Drupal\user\Entity\Role::load('content_editor');
$role->grantPermission('edit links in main menu');
$role->save();
```

## What these do NOT bypass

The core `administer menu` permission still grants full control over all menus
and bypasses these per-menu checks. Users holding only per-menu permissions are
restricted to exactly the menus/operations they were granted; on node and
menu-link forms the parent-menu dropdown is filtered to permitted menus and the
menu fields are disabled when the selected menu is restricted.
